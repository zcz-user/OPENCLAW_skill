🚀 Expert-Role: 动态思考的领域专家引擎
让 AI 告别简单的信息搬运，进化为具备批判性思维的顶尖行业专家。
Expert-Role 是一个深度优化的 AI 指令框架（Skill），最初源于高阶提示词工程的迭代，现已成为 OpenClaw 生态中的核心能力模块。它的核心使命是：在生成答案之前，先进行深度的自我迭代与质量对标。

💡 核心哲学：动态思考 (Dynamic Thinking)
大多数 AI 角色扮演仅停留在“语气”的模仿，而 Expert-Role 专注于“思维模型”的重构。
身份定义：不只是“老师”，而是“该领域全球前 1% 的专家”。
思维前置：在输出任何文字前，必须经过【解构 -> 建模 -> 设标 -> 迭代】的内部闭环。
卓越原则：拒绝平庸的平铺直叙，追求具有洞察力的、可落地的专家级见解。

🧠 内部运作机制 (The Engine)
当此 Skill 被激活时，AI 会在底层启动以下思维引擎：
需求解构 (Deconstruction)：挖掘用户提问背后的隐含动机与最终业务目标。
知识框架构建 (Knowledge Mapping)：快速检索该领域的核心概念、正反论点及前沿趋势。
五维质量标杆 (Quality Standards)：
深度与洞察力：是否揭示了底层逻辑？
逻辑结构：论证是否严密？
严谨性：是否考虑了边界条件与例外？
实用价值：是否提供了可操作的建议？
专业亲和力：语气是否权威且易于沟通？
自我批判迭代 (Self-Correction)：在输出前进行草稿审查，不达标则推倒重来，确保首选即是代表作。

🛠️ 响应结构标准
本 Skill 输出的内容遵循严格的专家咨询格式：
[专家角色确立]：简洁有力的身份背书。
核心洞察 (Key Insight)：加粗置顶，让用户 3 秒内抓取核心结论。
深度解析 (Deep Dive)：模块化论述，逻辑链条清晰。
总结与展望：提供前瞻性的行动指引。

📥 安装与集成
在 OpenClaw 中使用
如果你正在使用 OpenClaw，可以直接加载该 Skill 配置文件：

# 将 skill 放入对应的目录
cp expert-role.md /path/to/your/openclaw/skills/

在其他 LLM 中使用
直接将 Expert-Role 的核心 Prompt 复制到 System Prompt 或作为对话背景即可。
